# 新文章一键同步指南

## 日常使用

1. 在微信中打开“百味鸡 OB Pluto”的新文章，点击右上角并选择“复制链接”。
2. 在 Finder 中双击仓库根目录的 `一键同步.command`。
3. 链接通常会自动出现在输入框；点击“开始同步”。
4. 看到“同步完成：GitHub 已更新”即表示成功。

同步程序会自动完成：

- 识别普通图文或微信短内容；
- 转换正文为 Markdown；
- 下载并本地化正文图片；
- 追加稳定归档编号；
- 更新 `data/manifest.json`、`data/assets.json` 和 `README.md`；
- 运行完整性校验；
- 提交并推送到 GitHub 的 `main` 分支。

## 安全边界

- 只接受 `mp.weixin.qq.com` 文章链接。
- 只归档公众号名称为“百味鸡OB Pluto”的文章，防止误收录其他账号内容。
- 已归档的链接不会重复写入。
- 仓库存在未提交修改时会停止，避免覆盖人工改动。
- 任一图片下载或完整性校验失败时不会发布不完整版本。
- 不使用或保存公众号 `AppSecret`、Cookie、登录凭据。

## 日志与失败恢复

本地运行日志保存在 `.sync/logs/`，该目录不会提交到公开仓库。

如果正文或图片抓取失败，重新复制文章链接并再运行一次即可。如果本地提交已完成但网络导致 GitHub 推送失败，再次运行同一链接时会尝试补推此前的提交。

## 命令行检查

只检查链接和解析结果、不写入仓库：

```bash
python3 tools/sync_wechat.py "微信文章链接" --check
```

校验整个归档：

```bash
python3 tools/validate_archive.py
```
